02

Objective

Problem

Farmers in Kerala face literacy barriers, limited Malayalam-first tools, and delayed access to certified agricultural advice — especially for crop disease, seasonal planting, and market timing.

→

Solution

YieldUP is a multimodal AI advisory platform: speak in Malayalam, snap a crop photo, or type a query — get localized, context-aware guidance with escalation to human officers when needed.

Three stakeholders

🌾

Farmer

Voice-first queries in Malayalam, disease diagnosis from camera, regional crop and weather context.

👨‍🌾

Agri Officer

Dashboard for escalations, expert responses, and monitoring complex cases the AI cannot resolve.

📊

System / Ops

Docker-orchestrated services, Prometheus/Grafana observability, feedback loop for continuous model improvement.

04

Features

🎙

Malayalam Voice I/O

IndicSeamless STT with noise reduction and 10s chunking; Piper TTS for natural spoken responses — no cloud speech APIs.

📷

Crop Disease Detection

YOLOv11n nano model for real-time disease classification from farmer-uploaded crop images with minimal compute footprint.

🧠

RAG Advisory

Qdrant vector search over ICAR/KVK research docs + local Qwen3-8B for accurate, localized agricultural Q&A.

📈

Market & Planting Recs

scikit-learn / XGBoost engine synthesizes weather, crop history, and market signals with SHAP explainability.

🚨

Officer Escalation

Complex queries route to human agri-officers via dashboard; MongoDB stores escalation tickets and expert responses.

📡

Observability

Prometheus scrapes GPU/CPU/memory/API latency; Grafana dashboards monitor the full Krishi ecosystem health.

05

Workflows

  1. 1 Farmer speaks in Malayalam on Android app
  2. 2 Audio sent to FastAPI → IndicSeamless transcribes (denoise + 10s chunks)
  3. 3 Text query hits RAG pipeline: Qdrant retrieves ICAR/KVK context
  4. 4 Local Qwen3-8B generates localized advice
  5. 5 Piper TTS speaks response back to farmer
  1. 1 Farmer photographs affected crop leaf or soil
  2. 2 Image uploaded via FastAPI multipart endpoint
  3. 3 YOLOv11n classifies disease type in real time
  4. 4 RAG enriches diagnosis with treatment docs from Qdrant
  5. 5 Actionable remedy returned in Malayalam (voice or text)
  1. 1 AI confidence below threshold or farmer requests human help
  2. 2 Escalation ticket created in MongoDB with full query context
  3. 3 Agri officer sees ticket on dashboard with crop/region/season data
  4. 4 Officer responds with expert advice
  5. 5 Response delivered to farmer; feedback logged for model improvement

07

Challenges & Fixes

Problem

Field audio has background noise and Malayalam dialect variation

Fix

noisereduce preprocessing + 10s chunk processing in IndicSeamless; float16 + device_map for GPU memory

Problem

Cloud LLM (Gemini) — cost per query & data privacy concerns

Fix

Migrated to local Qwen3-8B via Ollama — zero API cost, on-prem inference, offline-capable

Problem

Heavy ASR model exhausts GPU VRAM on long recordings

Fix

torch.float16 precision, device_map="auto" CPU/GPU split, async background jobs for large files

Problem

Large audio files timeout on synchronous endpoints

Fix

Dual endpoints: /transcribe-sync for short clips, /transcribe async with job status polling

Problem

Diverse data types — structured accounts, chat logs, vector embeddings

Fix

Polyglot persistence: PostgreSQL + MongoDB + Qdrant + Redis, orchestrated via Docker Compose

08

Analytics & Cost Optimization

$0 Per-token LLM cost Local Qwen3-8B via Ollama
FP16 Inference precision Half VRAM for ASR + vision
10s Audio chunk size Prevents OOM on long recordings
Redis Market rate cache Reduces DB load on hot data
~10 Docker services Consistent dev/prod via Compose
0 Cloud speech API calls Whisper STT + Piper TTS local

Cost-aware edge AI: local models, free/open APIs where possible, aggressive caching, and async job processing — built for rural connectivity and farmer data privacy.
